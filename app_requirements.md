# Project Overview
Build a full-stack Webboard (forum) application. You act as an autonomous software engineer. Follow the architecture and requirements exactly as specified. 

# Tech Stack
- Frontend: React (Vite)
- Backend: Node.js with Express.js
- Database: PostgreSQL (Hosted on Supabase)
- ORM: Drizzle ORM
- Authentication: JWT (JSON Web Tokens) stored securely in HttpOnly cookies.

# Monorepo Structure & Configuration
Set up a root directory containing two main folders: `/frontend` and `/backend`.
**Strict Security Requirement:** The backend must use `cookie-parser`. CORS must be configured to strictly accept requests from the React frontend's origin and MUST have `credentials: true` enabled so the browser can send and receive the HttpOnly cookies.

# Database Schema (Drizzle ORM)
Create the following tables with strict referential integrity. 

**1. users**
- `id`: UUID (Primary Key, default random)
- `username`: varchar (Unique, Not Null)
- `password_hash`: varchar (Not Null)
- `created_at`: timestamp (default now)

**2. rooms**
- `id`: serial (Primary Key)
- `name`: varchar (Unique, Not Null)
- `description`: text

**3. posts**
- `id`: UUID (Primary Key, default random)
- `user_id`: UUID (Foreign Key to users.id, ON DELETE CASCADE)
- `room_id`: integer (Foreign Key to rooms.id, Not Null)
- `title`: varchar (Not Null)
- `content`: text (Not Null - will store Rich Text HTML)
- `views_count`: integer (Default 0)
- `tags`: text array (Optional)
- `created_at`: timestamp (default now)

**4. replies**
- `id`: UUID (Primary Key, default random)
- `post_id`: UUID (Foreign Key to posts.id, ON DELETE CASCADE)
- `user_id`: UUID (Foreign Key to users.id, ON DELETE CASCADE)
- `content`: text (Not Null)
- `created_at`: timestamp (default now)

# Core Backend Features & API Endpoints

**1. Authentication (HttpOnly Cookies)**
- `POST /api/auth/signup`: Accepts username and password. Hashes password, saves user. Generates a JWT and attaches it to the response as a secure, HttpOnly cookie.
- `POST /api/auth/login`: Verifies credentials. Generates a JWT and attaches it to the response as a secure, HttpOnly cookie.
- `POST /api/auth/logout`: Clears the HttpOnly cookie.
- `GET /api/auth/me`: Reads the HttpOnly cookie, verifies the JWT, and returns the current user's data (used by React to check auth state on page load).

**2. Rooms**
- `GET /api/rooms`: Fetches all 5 available rooms.

**3. Posts**
- `POST /api/posts`: Creates a new post. Middleware must verify the JWT from the HttpOnly cookie.
- `GET /api/posts`: Fetches posts. Must support SQL-level query parameters for filtering and searching:
  - `?roomId=X` (Filter by room)
  - `?search=keyword` (Search inside post title or content using ILIKE or full-text search)
- `GET /api/posts/:id`: Fetches a single post by ID and increments its `views_count`.
- `DELETE /api/posts/:id`: Deletes a post. *Strict requirement:* The API must verify that the `user_id` from the Auth JWT matches the `user_id` of the post before allowing deletion. 

**4. Replies**
- `GET /api/posts/:postId/replies`: Fetches the entire reply history for a specific post, ordered by oldest to newest.
- `POST /api/posts/:postId/replies`: Adds a reply to a post. Middleware must verify the JWT from the HttpOnly cookie.

# Frontend Behavior: Rich Text Editor
When creating a post or a reply, the React frontend MUST use a WYSIWYG Rich Text Editor (e.g., React Quill). 
- Users must be able to highlight text and click a "B" button to make it bold.
- Users must be able to click an "Image" icon to paste a direct image URL. The editor should embed this URL as a standard HTML `<img>` tag within the text.
- The raw HTML string generated by this editor is what gets sent to the backend and saved in the `content` column.
